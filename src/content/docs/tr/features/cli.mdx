Tauri, uygulamanızın sağlam bir komut satırı argüman ayrıştırıcısı olan [clap](https://github.com/clap-rs/clap) aracılığıyla bir CLI'ye sahip olmasını sağlar. `tauri.conf.json`  dosyanızdaki basit bir CLI tanımı ile arayüzünüzü tanımlayabilir ve JavaScript ve/veya Rust üzerindeki argüman eşleşmelerini okuyabilirsiniz.

## Desteklenen Platformlar

- Windows
- Linux
- macOS

## Kurulum

_Bu eklenti en az **1.75**_ Rust sürümü gerektirir

tauri-plugin-cli = "2.0.0-beta"

tauri-plugin-cli = { git = "https://github.com/tauri-apps/plugins-workspace", branch = "v2" }

```rust title="src-tauri/src/lib.rs" ins={4}
#[cfg_attr(mobile, tauri::mobile_entry_point)]
fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_cli::init())
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

## Temel Konfigürasyon

Arayüzü yapılandırmak için `tauri.conf.json` altında aşağıdaki yapıya sahipsiniz:

```json title="src-tauri/tauri.conf.json"
{
	"plugins": {
		"cli": {
			"description": "Tauri CLI Plugin Example",
			"args": [
				{
					"short": "v",
					"name": "verbose",
					"description": "Verbosity level"
				}
			],
			"subcommands": {
				"run": {
					"description": "Run the application",
					"args": [
						{
							"name": "debug",
							"description": "Run application in debug mode"
						},
						{
							"name": "release",
							"description": "Run application in release mode"
						}
					]
				}
			}
		}
	}
}
```

:::note

Buradaki tüm JSON konfigürasyonları sadece örnektir, diğer birçok alan açıklık sağlamak için atlanmıştır.

:::

## Adding Arguments

`args` dizisi, komutu veya alt komutu tarafından kabul edilen argümanların listesini temsil eder.

{/* TODO: List available configuration */}

### Positional Arguments

Konumsal bir argüman, argüman listesindeki konumu ile tanımlanır. Aşağıdaki yapılandırma ile:

```json title="src-tauri/tauri.conf.json"
{
	"args": [
		{
			"name": "source",
			"index": 1,
			"takesValue": true
		},
		{
			"name": "destination",
			"index": 2,
			"takesValue": true
		}
	]
}
```
Kullanıcılar uygulamanızı `./app tauri.txt dest.txt` olarak çalıştırabilir ve arg eşleşme haritası `source`'u `"tauri.txt"` ve  `destination`'i `"dest.txt"` olarak tanımlayacaktır.

### Named Arguments

Adlandırılmış bir bağımsız değişken, anahtarın değeri tanımladığı bir (anahtar, değer) çiftidir. Aşağıdaki konfigürasyonla:

```json title="tauri-src/tauri.conf.json"
{
	"args": [
		{
			"name": "type",
			"short": "t",
			"takesValue": true,
			"multiple": true,
			"possibleValues": ["foo", "bar"]
		}
	]
}
```

Kullanıcılar uygulamanızı `./app --type foo bar`, `./app -t foo -t bar` veya `./app --type=foo,bar` olarak çalıştırabilir ve arg eşleşmeleri haritası `type`'ı `["foo", "bar"]` olarak tanımlayacaktır.

### Flag Arguments

Bayrak argümanı, varlığı veya yokluğu uygulamanıza bilgi sağlayan bağımsız bir anahtardır. Aşağıdaki konfigürasyonla:

```json title="tauri-src/tauri.conf.json"
{
	"args": [
		{
			"name": "verbose",
			"short": "v"
		}
	]
}
```
Kullanıcılar uygulamanızı `./app -v -v -v`, `./app --verbose --verbose --verbose` veya `./app -vvv` şeklinde çalıştırabilir ve arg eşleşmeleri haritası `verbose` öğesini `true` olarak ve `occurrences = 3` ile tanımlayacaktır.

## Alt Komutlar

Bazı CLI uygulamaları alt komutlar olarak ek arayüzlere sahiptir. Örneğin, `git` CLI'sı `git branch`, `git commit` ve `git push`a sahiptir. Ek iç içe arayüzleri `subcommands` dizisi ile tanımlayabilirsiniz:

```json title="tauri-src/tauri.conf.json"
{
  "cli": {
    ...
    "subcommands": {
      "branch": {
        "args": []
      },
      "push": {
        "args": []
      }
    }
  }
}
```

Yapılandırması, `description`, `longDescription`, `args` vb. ile kök uygulama yapılandırmasıyla aynıdır.

## Kullanım

CLI eklentisi hem JavaScript hem de Rust dillerinde kullanılabilir.

<Tabs>
  <TabItem label="JavaScript">

```js
import { getMatches } from '@tauri-apps/plugin-cli';

const matches = await getMatches();
if (matches.subcommand?.name === 'run') {
	// `./your-app run $ARGS` was executed
	const args = matches.subcommand.matches.args;
	if (args.debug?.value === true) {
		// `./your-app run --debug` was executed
	}
	if (args.release?.value === true) {
		// `./your-app run --release` was executed
	}
}
```

  </TabItem>
  <TabItem label="Rust">

```rust title="src-tauri/src/lib.rs" {1, 6-18}
use tauri_plugin_cli::CliExt;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
fn run() {
   tauri::Builder::default()
       .plugin(tauri_plugin_cli::init())
       .setup(|app| {
           match app.cli().matches() {
               // `matches` here is a Struct with { args, subcommand }.
               // `args` is `HashMap<String, ArgData>` where `ArgData` is a struct with { value, occurrences }.
               // `subcommand` is `Option<Box<SubcommandMatches>>` where `SubcommandMatches` is a struct with { name, matches }.
               Ok(matches) => {
                   println!("{:?}", matches)
               }
               Err(_) => {}
           }
           Ok(())
       })
       .run(tauri::generate_context!())
       .expect("error while running tauri application");
}
```

  </TabItem>
</Tabs>

## İzinler

Varsayılan olarak tüm eklenti komutları engellenir ve erişilemez.
`capabilities` yapılandırmanızda bir izinler listesi tanımlamalısınız.

Daha fazla bilgi için [Erişim Kontrol Listesi](/references/v2/acl) bölümüne bakınız.

```json title="src-tauri/capabilities/main.json" ins={6}
{
	"$schema": "./schemas/desktop-schema.json",
	"identifier": "main-capability",
	"description": "Capability for the main window",
	"windows": ["main"],
	"permissions": ["cli:default"]
}
```

| İzin 					  | Açıklama 																		|
| ----------------------- | ------------------------------------------------------------------------------- |
| `cli:default` 		  | CLI eşleşmelerinin okunmasına izin verir.                                   	|
| `cli:allow-cli-matches` | Önceden yapılandırılmış bir kapsam olmadan cli_matches komutunu etkinleştirir.  |
| `cli:deny-cli-matches`  | Önceden yapılandırılmış bir kapsam olmadan cli_matches komutunu reddeder.  		|
