[Qwik](https://qwik.builder.io/) ist ein neuartiges Web-Framework,
das schnell ladende Webanwendungen beliebiger Größe und Komplexität unterstützt.
Es gibt eine Qwik-React Integration, mit der Sie die Möglichkeit haben Ihre React Komponenten migrieren zu können.
Erfahren Sie mehr über Qwik unter https://qwik.builder.io und in den [Dokumentationen](https://qwik.builder.io/docs/).

## Integration in vorhandenes Projekt


### Erstelle eine Qwik Anwendung

Falls noch keine Qwik Anwendung vorhanden ist, können Sie eine wie folgt erstellen:

<Tabs>
<TabItem label="npm">
```sh

```
</TabItem>

<TabItem label="pnpm">
```sh
pnpm create qwik@latest
```
</TabItem>
</Tabs>


1. _Project name_  
   Dies ist der Name Ihres JavaScript-Projekts. Er entspricht dem Namen des Ordners, den dieses Dienstprogramm erstellt, hat aber ansonsten keine Auswirkungen auf Ihre Anwendung. Sie können hier jeden beliebigen Namen verwenden.

2. _Select a starter_  
   Wenn Sie möchten wählen Sie die `Basic App ((Demo App with Routing built-in)` als Beispiel Vorlage aus.

3. _Install dependencies_  
   Ja, wenn Sie die Abhängigkeiten automatisch installieren möchten.

4. _Initialize git_  
   Ja, wenn Sie Ihr Projekt als neues Git-Repository vorbereiten möchten.

## Aufrufen von Rust-Befehlen

Mit Tauri können Sie Ihr Frontend um native Funktionen erweitern.
Wir nennen diese Befehle, im Wesentlichen Rust-Funktionen, die Sie von Ihrem Frontend-JavaScript aus aufrufen können.
Dadurch können Sie schwere Verarbeitungen oder Aufrufe an das Betriebssystem in viel performanterem Rust-Code verarbeiten.

Machen wir ein einfaches Beispiel ...

### Erstelles eines command

Ein command ist wie jede normale Rust-Funktion,
mit dem Zusatz des Attributs #[tauri::command] Makro,
das Ihrer Funktion die Kommunikation mit dem
dem JavaScript-Kontext zu kommunizieren ermöglicht.

```rs title="src-tauri/src/lib.rs"
#[tauri::command]
fn greet(name: &str) -> String {
   format!("Hello, {}!", name)
}
```

Zu guter Letzt müssen wir Tauri auch über unseren neu erstellten
Kommando informieren, damit es die Aufrufe entsprechend weiterleiten kann.
Dies geschieht mit der Kombination aus der Funktion _.invoke_handler()_
Funktion und dem Makro _generate_handler![]_:

```rs title="src-tauri/src/lib.rs"
#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
  tauri::Builder::default()
    .invoke_handler(tauri::generate_handler![greet])
    .run(tauri::generate_context!())
    .expect("error while running tauri application");
}
```

### Erstellen einer Qwik Komponente

Um unseren neu erstellten Befehl aufzurufen, verwenden wir
die JavaScript-Bibliothek _@tauri-apps/api_.
Sie können sie mit Ihrem bevorzugten JavaScript-Paketmanager
installieren (siehe erster [Schritt](/de/guides/frontend/qwik#installieren-sie-das-tauri-api-package)).
Sie ermöglicht den Zugriff auf Kernfunktionen wie windows,
das Dateisystem und mehr durch praktische JavaScript
Abstraktionen.

Nachdem die Bibliothek installiert ist, können wir nun eine neue Qwik-Komponente erstellen:

```sh
mkdir src/components/greet
```

```tsx title="src/components/greet/greet.tsx"
import { $, component$, useSignal } from "@builder.io/qwik";
import { invoke } from "@tauri-apps/api/core";

export default component$(() => {
  const greetMsg = useSignal("");

  const greet = $(async (name: string) => {
    greetMsg.value = await invoke("greet", { name });
  });

  return (
    <div>
      <button onClick$={() => greet("Qwik")}>Greet</button>
      <p>{greetMsg.value}</p>
    </div>
  );
});
```

Sie können Ihre Komponente nun anwenden zum Beispiel in der _src/routes/index.tsx_:

```tsx title="src/routes/index.tsx"
import { component$ } from "@builder.io/qwik";
import type { DocumentHead } from "@builder.io/qwik-city";
import Greet from "~/components/greet/greet";

export default component$(() => {
  return <Greet />;
});

export const head: DocumentHead = {
  title: "Welcome to Qwik",
  meta: [
    {
      name: "description",
      content: "Qwik site description",
    },
  ],
};
```
